    【Vue】基础总结(持续更新)

    谦逊 5/10/2022 Vue基础

    # Vue总结

    # Vue-DevTools插件

    拓展功能,让浏览器可以运行Vue

    # 模板语法

    - 1.插值语法: {{}} 
    xxx是js表达式
    绑定data中的数据,也可以调用函数、携带参数
- 2.指令语法
    v-xx:
    v-bind  单向数据绑定
    v-model 双向数据绑定 => 就是取input的value值
    ...

    # 数据绑定

    - 1.单向数据绑定
    数据只能通过data改变页面
- 2.双向数据绑定
    data与页面数据互相改变

    # el与data的写法

    - 1.el两种写法:
    - 1.1 new Vue时 => el:''
    - 1.2 挂载时 => $mount('#xx')
- 2.data两种写法:
    - 2.1 对象式
        data:{

        }
    - 2.2 函数式  => 组件写法
        data() {
            return {
                
            }
        },

    注意:Vue所管理的函数要写成普通函数,不是Vue所管理的函数要写成箭头函数

    # MVVM模型

    - Vue框架的作用就是将数据通过new Vue实例反应到View界面

    # 数据代理

    - 原理: Object.defineProperty(目标对象,'属性或方法'),{
        - value: 值,
        - writable: 能否修改值,
        - enumerable: 能否被遍历,
        - configurable: 能否被delete
}
- 重点:
    - getter、setter函数通过目标对象监听另一个对象的值是否改变
- Vue中的数据代理:
    - 通过new Vue实例来代理data对象中的属性的操作(读/写)
- 基本原理:
    - Vue为每个添加到Vue实例上的属性,绑定一个getter、setter函数进行读写操作
- 注意: data是由$options包裹

    # 事件处理

    - v-on:xxx或@xxx绑定事件,可以传参
- 事件修饰符:
    - .stop    阻止冒泡,默认冒泡 (常用)
    - .prevent 阻止默认行为      (常用)
    - .once    事件只触发一次    (常用)
    - .capture 使用事件捕获,默认不捕获
    - .self    event.target是当前操作元素才触发
    - .passive 立即执行事件默认行为 
    - .native  原生js事件
- 常见键盘事件
    - enter 
    - delete
    - esc
    - space
    - tab
    - up
    - down
    - left
    - right
    - eg: @click.enter='XXX'

    # Computed计算属性

    - get和set函数 => 一般只用简写形式set函数
- get函数调用时机:
- 2.1 第一次模板解析时
- 2.2 当所需的值变化时,重新计算
- 必须要有返回值

    注意:Computed的函数调用时机 => 导致不能使用定时器函数,因为所需值没有变化

    # watch监视属性

    - 使用注意:
    - 属性存在时生效
    - 当监视的属性变化时,调用该函数
- API
    - deep:true => 深度监视 => 适用于监视目标为对象时
    - immediate:true => 模板解析时调用一次该函数
    - handler(newvalue,oldvalue) => 一般使用简写形式
- 简写形式:
    -  xxx(newvalue,oldvalue){
        console.log('isHot监视中 => ', newValue, oldValue);
    }
- 使用方式:
    - 1) new Vue实例时
    - 2) 创建Vue实例后,通过$watch('xxx',{})挂载

    # 绑定样式

    - 绑定class样式
    - 1. 字符串写法 => 适用于类名不确定,需要动态指定(通过随机数)
    - 2. 数组写法 => 适用于绑定样式数量不确定、名字也不确定 
    - 3. 对象写法 => 适用于样式数量确定、名字也确定,但需要动态指定样式
- 绑定style样式
    - 1. :style="{fontSize:xxx}" 其中xxx是动态值
    - 2. :style="[a,b]",其中a,b是对象形式

    # 条件渲染

    - v-show
    - display:none实现,结构还在
    - 适用于切换频率较高的场景
- v-if
    - 会破坏原有结构
    - 适用于切换频率较低的场景
- v-else-if
    - 除去v-if之后的依次判断
- v-else
    - 除去之前所有的判断

    注意:v-if可能会获取不到样式 => 因为结构会消失找不到该元素

    # 列表渲染

    - v-for
    - 写法:
        - v-for:"ele in xxx" 
    - 场景
        - 遍历数组
            - v-for="ele in Array"
        - 遍历对象
            - v-for="ele in Object"
        - 遍历字符串(少用)
            - v-for="ele in String"
    - 使用注意
        - v-for(value,key) in Object => value在前,key在后
- key的作用
    - key给每个列表添加一个唯一标识,方便Diff算法识别
    - Diff算法简易过程
        - 判断key值 => 判断元素 => 判断value值 ...
        - 生成虚拟DOM / 复用虚拟DOM
- 列表过滤
    - 使用computed计算属性最好 => 记住必须要有返回值
    - 搭配filter / forEach / indexOf 使用
- 列表排序
    - 通过一个flag值,选择排序方式
    - 最后进入函数判断
    - 搭配三元运算符 / sort 使用

    注意:使用v-for时,必须添加上:key值 => 通常使用nanoid / uuid(nanoid的缩小版)

    # 数据更新 / 插入问题

    - 使用前提必须是响应式数据
- Vue.set() / vm.$set() / this.$set() 可以直接给data中的数据添加
- 1.1 数组 => 使用不破坏原数组的方法pop、push、shift、unshift等 
=> 原因:Vue将Array上的方法放在了Vue实例上
- 1.2 对象 => 直接使用Vue.set(target,key,value)注意不要给data根对象添加属性

    # 收集表单数据

    - 通过双向绑定数据进行存储
- 获取数据后需要转换为JSON格式

    # 过滤器

    - 使用
    - xxx | xxx
- 只能在Vue2中使用
- 常用于文本格式化
- 定义方法:
    - 局部 
        - filters:{
            xxx(){
                return xxx
            }
        }
    - 全局
        - Vue.filter('xxx',function(){
            return xxx
        })

    # 内置指令

    - v-text  获取文本
- v-html  不推荐使用 => 会影响xss攻击
- v-cloak 配合css解决出现{{}}的问题
- v-once  只渲染一次,优化性能
- v-pre   不需要经过Vue模板解析

    # 自定义指令

    - 局部
    - new Vue({
        directives:{
           xxx:{

           } 
        }
    })
    或
    - new Vue({
        directives(){

        }
    })
- 全局指令:
    - Vue.directive(指令名,配置对象)或 Vue.directive(指令名,回调函数)
- 配置对象中的三个回调函数  
    - bind      绑定时
    - inserted  插入页面时
    - update    重新解析时
