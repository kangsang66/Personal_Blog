    【Vue Router】基本使用

    谦逊 5/31/2022 Vue进阶

    # 1. Vue-Router 路由

    # 1.1 作用:

    实现单页面应用,优化用户体验,减少跳转时间

    # 1.2 用法:

    1. 创建router文件夹,以及index文件

    2. 创建并暴露 通过VueRouter new出来的构造函数

    Router / index.js

    // 导入VueRouter
import VueRouter from 'vue-router'

// 导入路由所需模块
import Home from '../components/Home'
import About from '../components/About'
import Info from '../components/Info'

// 通过VueRouter构造一个函数
export default new VueRouter({
    // 注意这里必须使用routes包含
    // 写成数组形式
    // path、component
    routes:[
        {
            path:'/Home',
            component:Home
        },
        {
            path:'/About',
            component:About
        },
        {
            path:'/Info',
            component:Info
        }
    ]
})
    1. 引入暴露出来的VueRouter构造函数

      index.js

    import Vue from 'vue'
import App from './App.vue'
// 引入Store外部文件
// 导入vue-router
import VueRouter from 'vue-router'
import router from './router'

// 使用VueRouter
Vue.use(VueRouter)
Vue.config.productionTip = false

new Vue({
  render: h => h(App),
  // 直接使用router
  router
}).$mount('#app')

    # 1.3 声明式路由:

    <router-link :to="...">

    标签:

    <router-link><router-link>
<router-view><router-view>

    <!-- router点击链接标签 => 类似a标签 -->
<router-link class="red" active-class="active" to="/Home">Home</router-link>
<router-link class="red" active-class="active" to="/About">About</router-link>

    # 1.4 注意:

    • pages文件夹放路由组件 / components文件夹放一般组件
    • 路由组件的切换,未使用的组件会被销毁,而不是display:none => 组件缓存问题
    • 每个路由组件都有$route / $router,其中$route包含的信息是不同的

    # 2. 嵌套路由用法:

    - 直接给一级路由添加children配置项
- 不能给嵌套路由添加 / 
- 注意嵌套路由路径需要加上父亲路由路径

        children:[
        {
            path:'xxx',
            component:xxx
        },
        {
            path:'xxx',
            component:xxx
        }
    ]

        <router-link to="/home/firstChild" class="list-group-item" active-class="active">点击进入FirstChild</router-link>

    <router-link to="/about/secondChild" class="list-group-item" active-class="active">点击进入SecondChild</router-link>

    # 2.1 query参数

    # 作用:

    复用组件,给组件传递参数

    字符串写法:

    注意:
1. 必须写成 : 数据绑定形式 => 以js代码进行编译
2. 模板字符串加变量

        <router-link :to="`/home/firstChild/tab?id=${ele.id}&${ele.value}`">{{ele.value}}</router-link>

    对象写法:

    注意:
1. path和query参数
2. query为对象写法

        <router-link :to="{
        path:'/home/firstChild/tab',
        query:{
            id:ele.id,
            value:ele.value
        }
    }">
        {{ele.value}}
    </router-link>

    # 2.2 命名路由

    给路由配置项添加name
简化对象写法中的path路径

    配置:

        {
        name:'tab',  // 命名
        path:'tab',
        component:Tab
    }

    使用:

        <router-link :to="{
        name:'tab',  // 使用命名
        query:{
            id:ele.id,
            value:ele.value
        }
    }">
        {{ele.value}}
    </router-link>

    # 2.3 params参数

    - 注意点
    - 类似query参数
    - 只能通过name表示路径,不能使用path表示

        <router-link :to="{
        name:'tab',  // 使用命名
        params:{
            id:ele.id,
            value:ele.value
        }
    }">
        {{ele.value}}
    </router-link>

        {
        path:'firstChild', 
        component:FirstChild,
        children:[ // /home/firstChild/tab/id/value
            {
                path:'tab/:id/:value', // 必须使用占位符 : 分隔
                component:Tab
            }
        ]
    },

    使用this.$route.params进行调用
    <h3>我是{{$route.params.value}}的内容</h3>

    # 2.4 query参数与params的区别:

    • query参数指定路径方式:name、path
    • params参数指定路径方式:name,给路由组件配置name

    # 2.5 路由的props配置

    • 让路由更方便的接收参数,使用props接收
    • 三种写法,推荐函数式写法
        {
        name:'tab',
        path:'tab/id/value',
        component:tab,
        // 第一种 对象写法(不推荐,有局限性)
        props:{
            id:'666',
            value:'你好啊'
        },
        // 第二种 布尔值(一般,只能接受params参数),需要配置name并使用
        props:true，
        // 第三种 函数式(推荐),简化前缀,返回对象,参数为$route
        props($route){
            id:$route.query.id / params.id,
            value:$route.query.value / params.value
        }
    }

    # 2.6 <router-link>的replace属性

    1. 作用:控制路由跳转时操作浏览器历史记录的模式
2. 浏览器的历史记录有两种写入方式:分别为push和  replace,push是追加历史记录,replace是替换当前记录, 路由跳转时候默认为push
3. 如何开启replace模式:\<router-link replace ...    >XXX\<router-link>

    # 3. 编程式路由导航

    1. 作用:解决router-link标签转换为a标签的局限性
2. 原理:Vue对于history浏览器历史记录的操作
3. API
this.$router.forward() // 前进
this.$router.back()    // 后退
this.$router.go(Num)   // 前进或后退多少,里面写入参数

    # 4. 缓存路由

    1. 作用:解决路由组件被销毁的问题
2. 用法:设置keep-Alive标签包裹路由展示组件
3. 注意:缓存路由组件会多出activated、deactivated两个新的生命周期函数
4. activated、deactivated

    // 路由组件激活时调用
activated(){
    console.log('当前组件展示中')
},
// 路由组件未激活时调用
deactivated(){
    console.log('当前组件未展示')
}

    4. 触发顺序:created => mouted => activated => deactivated
5. 缓存路由分类:
    - 5.1 全部缓存 
      添加上keep-alive标签

        <keep-alive>
        <router-view></router-view>
    </keep-alive>

        - 5.2 指定单个缓存  
      添加include="组件name"
      注意是当前组件的name,而不是路由配置的name

        <keep-alive inclue="name">
        <router-view></router-view>
    </keep-alive>

        - 5.3 指定多个缓存  
      配置路由meta对象属性并添加keepAlive : true
      使用v-if判断当前keepAlive属性值

        <keep-alive>
        <router-view v-if="$route.meta.keepAlive"></router-view>
    </keep-alive>
    <router-view v-if="!$route.meta.keepAlive"></router-view>

        {
        path:'FirstChild',
        component:Home,
        meta:{
            keepAlive:true
        }
    }

    # 全局路由守卫

    1. 作用:权限管理,进入组件前后doSomething
2. 用法:调用VueRouter创建的实例对象身上的方法
3. API:beforeEach(to,from,next)、afterEach(to,from)
4. 注意:afterEach没有next方法,
5. 调用时机:第一次渲染以及切换前后调用
6. 配置

    meta:{
    isAuth:true // 键为某个属性名
}

    // 全局前置路由守卫
router.beforeEach((to,from,next)=>{
    if(to.name === 'firstChild' || to.name === 'secondChild'){
        // 配合
        if(localStorage.name === 'kangzhen'){
            next()
        }else {
            alert('请先登录!')
        }
    }else {
        next()
    }
})
// 全局后置路由守卫
router.afterEach((to,from)=>{
    if(to.name === 'firstChild'){
        console.log('来到了firstChild组件');
    }
})

    省略写法:
// 全局前置路由守卫
router.beforeEach((to,from,next)=>{
    if(to.meta.isAuth){
        // 配合
        if(localStorage.name === 'kangzhen'){
            next()
        }else {
            alert('请先登录!')
        }
    }else {
        next()
    }
})
// 全局后置路由守卫
router.afterEach((to,from)=>{
    if(to.meta.isAuth){
        console.log('来到了firstChild组件');
    }
})

    # 组件内路由守卫

    1.组件内的路由守卫
2.beforeRouteEnter(to,from,next)、beforeRouteLeave(to,from,next)
3.通过路由规则时调用

    // 通过路由规则,进入该组件调用
beforeRouteEnter(to, from, next) {
  if(to.meta.isAuth){
      // 配合
      if(localStorage.name === 'kangzhen'){
          next()
      }else {
          alert('请先登录!')
      }
  }else {
      next()
  }
},
// 通过路由规则,离开该组件调用
beforeRouteLeave (to, from, next) {
  console.log('离开了About组件');
  next()
},

    # hash模式与history模式

    1.两者区别:
    1.1 形式上:
        hash   :  /#/
        history:  没有#
    1.2 兼容性上:
        hash      更好 例如:对于IE 6、7、8
        history   略差
    1.3 网络请求上:
        hash      不会向服务器请求资源
        history   会向服务器请求资源,导致 404 NOT FOUND     
2.用法:
    2.1 在new VueRouter配置添加mode:'hash / history'
    2.2 使用express框架,创建简易服务器,通过static部署上线
3.注意:
    3.1 如果必须使用hash模式,需要后端人员配合,写中间件或者路由,或者前端使用connect-history-api-fallback插件


    // 一个简易的服务器
const express = require('express');
const app = express();
// 这里使用express框架下的static部署静态资源
app.use(express.static(__dirname+'/dist'))
app.get('/personInfo',(response,request)=>{
    const data = {
        name:'康振',
        age:20,
        sex:'男'
    }
    request.send(JSON.stringify(data))
})

app.listen(5000,(error)=>{
    if(error){
        console.log('服务器启动失败');
    }else{
        console.log('服务器启动成功,端口号在http://localhost:5000');
    }
})
